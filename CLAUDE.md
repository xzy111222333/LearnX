# AI 辅助开发规则文件

## 前端开发规则

### 1. 代码风格

- **命名规范**：
  - 组件名：使用 PascalCase（如 `MaterialCard`）
  - 函数名：使用 camelCase（如 `handleUpload`）
  - 变量名：使用 camelCase（如 `isLoading`）
  - 常量名：使用 UPPER_SNAKE_CASE（如 `MAX_FILE_SIZE`）
  - 文件名：使用 kebab-case（如 `file-upload.tsx`）

- **代码格式**：
  - 使用 Prettier 进行代码格式化
  - 缩进使用 2 个空格
  - 每行代码长度不超过 80 字符
  - 使用单引号而非双引号
  - 大括号使用 ESLint 推荐风格

- **注释规范**：
  - 组件需要添加 JSDoc 注释，说明功能和参数
  - 复杂逻辑需要添加行内注释
  - 避免冗余注释，代码本身应清晰易懂

### 2. 组件开发

- **组件设计原则**：
  - 单一职责：每个组件只负责一个功能
  - 可复用性：设计通用组件，避免重复代码
  - 可测试性：组件逻辑应易于测试
  - 性能优化：避免不必要的渲染

- **组件结构**：
  - 原子组件：基础 UI 组件（按钮、输入框等）
  - 分子组件：功能组件（表单、卡片等）
  - 页面组件：完整页面（首页、详情页等）

- **Props 设计**：
  - 使用 TypeScript 类型定义 Props
  - 为可选 Props 设置默认值
  - 避免过多 Props，考虑使用配置对象

### 3. 状态管理

- **状态分层**：
  - 全局状态：使用 Context API 管理
  - 局部状态：使用 useState 和 useReducer
  - 表单状态：使用 React Hook Form

- **状态更新**：
  - 使用函数式更新避免闭包问题
  - 批量更新状态减少渲染
  - 避免不必要的状态更新

### 4. 路由管理

- **路由设计**：
  - 使用 Next.js App Router
  - 合理组织路由结构
  - 实现路由守卫保护敏感页面

- **导航**：
  - 使用 Link 组件进行客户端导航
  - 避免使用 window.location 进行页面跳转
  - 处理路由参数和查询参数

### 5. API 调用

- **API 封装**：
  - 统一封装 API 请求函数
  - 处理错误和加载状态
  - 添加请求超时和重试机制

- **数据处理**：
  - 使用 Zod 进行数据验证
  - 转换 API 响应数据格式
  - 缓存频繁请求的数据

### 6. 样式管理

- **样式方案**：
  - 使用 Tailwind CSS 进行样式开发
  - 避免使用内联样式
  - 合理使用自定义工具类

- **响应式设计**：
  - 使用 Tailwind 的响应式类
  - 为不同设备优化布局
  - 测试各种屏幕尺寸

### 7. 性能优化

- **渲染优化**：
  - 使用 React.memo 缓存组件
  - 使用 useMemo 缓存计算结果
  - 使用 useCallback 缓存函数

- **资源优化**：
  - 图片懒加载
  - 代码分割
  - 减少打包体积

- **网络优化**：
  - 合理使用缓存策略
  - 压缩 API 响应
  - 减少 HTTP 请求

### 8. 测试策略

- **测试类型**：
  - 单元测试：测试组件和函数
  - 集成测试：测试组件交互
  - E2E 测试：测试完整流程

- **测试工具**：
  - Jest：单元测试
  - React Testing Library：组件测试
  - Cypress：E2E 测试

### 9. 安全措施

- **前端安全**：
  - 输入验证，防止 XSS 攻击
  - 避免在前端存储敏感信息
  - 使用 HTTPS 传输数据

- **API 安全**：
  - 添加认证令牌
  - 防止 CSRF 攻击
  - 限制 API 访问频率

### 10. 开发流程

- **开发工具**：
  - VS Code：代码编辑器
  - ESLint：代码质量检查
  - Prettier：代码格式化
  - Git：版本控制

- **开发规范**：
  - 遵循 Git 工作流
  - 提交信息规范
  - 代码审查流程
  - 定期更新依赖

### 11. AI 辅助开发规范

- **AI 工具使用**：
  - 使用 AI 生成代码模板和示例
  - 利用 AI 进行代码审查和优化
  - 借助 AI 解决技术问题

- **AI 生成代码处理**：
  - 仔细审查 AI 生成的代码
  - 确保代码符合项目规范
  - 测试 AI 生成的功能
  - 理解代码逻辑后再使用

- **AI 提示词设计**：
  - 提供详细的上下文信息
  - 明确指定代码风格和规范
  - 要求生成测试用例
  - 提供具体的功能需求

### 12. 最佳实践

- **代码组织**：
  - 按功能模块组织代码
  - 合理使用文件夹结构
  - 保持代码结构清晰

- **文档管理**：
  - 维护组件文档
  - 更新 API 文档
  - 记录开发决策

- **团队协作**：
  - 统一代码风格
  - 定期代码审查
  - 分享开发经验

- **持续改进**：
  - 定期重构代码
  - 优化性能
  - 学习新技术

## 后端开发规则

### 1. 代码风格

- **命名规范**：
  - 类名：使用 PascalCase（如 `UserService`）
  - 函数名：使用 camelCase（如 `getUserById`）
  - 变量名：使用 camelCase（如 `userData`）
  - 常量名：使用 UPPER_SNAKE_CASE（如 `JWT_SECRET`）
  - 文件名：使用 kebab-case（如 `auth.controller.ts`）

- **代码格式**：
  - 使用 Prettier 进行代码格式化
  - 缩进使用 2 个空格
  - 每行代码长度不超过 80 字符
  - 使用单引号而非双引号
  - 大括号使用 ESLint 推荐风格

- **注释规范**：
  - 函数和类需要添加 JSDoc 注释，说明功能和参数
  - 复杂逻辑需要添加行内注释
  - 避免冗余注释，代码本身应清晰易懂

### 2. 项目结构

- **目录组织**：
  - `controllers/`：处理 HTTP 请求
  - `services/`：业务逻辑
  - `models/`：数据模型
  - `routes/`：路由定义
  - `middleware/`：中间件
  - `utils/`：工具函数
  - `config/`：配置文件

- **模块划分**：
  - 按功能模块划分（如认证、素材、订单、收益）
  - 每个模块包含对应的控制器、服务和路由
  - 保持模块间低耦合

### 3. 数据库设计

- **数据模型**：
  - 使用 Sequelize ORM 定义数据模型
  - 合理设计表结构和关联关系
  - 添加适当的索引优化查询性能

- **数据迁移**：
  - 使用 Sequelize 的同步功能自动创建表结构
  - 定期备份数据库
  - 保持数据一致性

### 4. API 设计

- **RESTful 规范**：
  - 使用 HTTP 方法：GET、POST、PUT、DELETE
  - 合理设计 API 路径（如 `/api/auth/login`）
  - 返回标准的 HTTP 状态码

- **响应格式**：
  - 统一响应格式，包含 status、message 和 data
  - 错误处理返回详细的错误信息
  - 成功响应返回相关数据

- **API 版本控制**：
  - 在路径中包含版本号（如 `/api/v1/auth/login`）
  - 保持 API 向后兼容

### 5. 认证与授权

- **认证机制**：
  - 使用 JWT 进行用户认证
  - 实现登录、注册和令牌刷新功能
  - 令牌过期时间合理设置

- **授权控制**：
  - 实现基于角色的权限控制
  - 验证用户是否有权限访问资源
  - 保护敏感 API 端点

### 6. 错误处理

- **错误类型**：
  - 客户端错误：4xx 状态码
  - 服务器错误：5xx 状态码
  - 业务逻辑错误：自定义错误处理

- **错误处理中间件**：
  - 统一捕获和处理错误
  - 记录错误日志
  - 返回友好的错误信息

### 7. 安全性

- **数据安全**：
  - 密码加密存储
  - 防止 SQL 注入
  - 输入验证和清理

- **API 安全**：
  - CORS 配置
  - 防止 CSRF 攻击
  - 限制 API 访问频率

- **环境变量**：
  - 敏感配置使用环境变量
  - 避免硬编码密钥和密码
  - 不同环境使用不同配置

### 8. 性能优化

- **数据库优化**：
  - 合理使用索引
  - 优化查询语句
  - 避免 N+1 查询问题

- **服务器优化**：
  - 使用缓存减少数据库查询
  - 合理使用异步处理
  - 优化文件上传和处理

- **代码优化**：
  - 减少重复代码
  - 优化算法和数据结构
  - 定期代码重构

### 9. 测试策略

- **测试类型**：
  - 单元测试：测试函数和方法
  - 集成测试：测试模块间交互
  - API 测试：测试 API 端点

- **测试工具**：
  - Jest：单元测试
  - Supertest：API 测试
  - MongoDB Memory Server：数据库测试

### 10. 部署与监控

- **部署策略**：
  - 使用 PM2 管理进程
  - 配置环境变量
  - 定期备份数据

- **监控与日志**：
  - 记录 API 请求和错误
  - 使用日志管理工具
  - 监控服务器性能

### 11. AI 辅助开发规范

- **AI 工具使用**：
  - 使用 AI 生成代码模板和示例
  - 利用 AI 进行代码审查和优化
  - 借助 AI 解决技术问题

- **AI 生成代码处理**：
  - 仔细审查 AI 生成的代码
  - 确保代码符合项目规范
  - 测试 AI 生成的功能
  - 理解代码逻辑后再使用

- **AI 提示词设计**：
  - 提供详细的上下文信息
  - 明确指定代码风格和规范
  - 要求生成测试用例
  - 提供具体的功能需求
